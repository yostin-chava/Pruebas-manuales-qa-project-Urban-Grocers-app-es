# Prueba 3

## Descripción de la prueba

Este documento describe las actividades realizadas para la validación del formulario de reserva, las funcionalidades relacionadas y el diseño de la interfaz.

### 1. Verificación del diseño del formulario de reserva

- Se crea una lista de comprobación a partir de los requisitos funcionales.
- Se analiza el diseño en Figma para verificar alineación con los requisitos.

### 2. Verificación de ventanas "Método de pago" y "Agregar tarjeta"

- Se crea una lista de comprobación basada en:
  - Partición de clases de equivalencia.
  - Análisis de valores límite.

### 3. Casos de prueba para el botón "Reservar"

- Se preparan y documentan diferentes escenarios de uso del botón "Reservar".

### 4. Casos de prueba para la función de alquiler de automóviles

- Se diseñan casos de prueba que cubren múltiples combinaciones de entrada y resultados esperados.

### 5. Ejecución de pruebas y reporte de errores

- Las pruebas de diseño de la aplicación se realizan en dos entornos:

  - **Google Chrome** – Resolución: 800x600  
  - **Mozilla Firefox** – Resolución: 1920x1080

- Los errores encontrados se reportan en Jira.
